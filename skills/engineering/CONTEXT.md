# engineering — Catalog Context

Rules, notes, and references that apply only to skills in this catalog.
(Repo-wide standards live in `.agents/knowledge/skill-quality.md`.)

## Requirements

- Methodology level only: a skill here must describe *how to approach* a
  class of engineering problems, not how to operate one language, framework,
  or tool. If the instructions would change when the tech stack changes, the
  skill does not belong here.
- Examples inside a skill may use a concrete language for illustration, but
  the guidance itself must transfer across stacks.
